# FitClub — Сервис фитнес-клуба

> Автоматизированная система управления абонементами, тренировками и посещаемостью фитнес-клуба.

## О проекте

Проект представляет собой веб-приложение для автоматизации ключевых процессов фитнес-клуба. Оно обеспечивает взаимодействие трёх основных ролей: клиентов, тренеров и администраторов, позволяя удобно управлять расписанием, абонементами и отслеживать посещаемость.

Ключевые возможности:
- **Для клиентов**: онлайн-запись на тренировки, покупка и управление абонементами, просмотр истории посещений и отслеживание статистики.
- **Для тренеров**: просмотр личного расписания тренировок и управление посещаемостью клиентов на своих занятиях.
- **Для администраторов**: управление расписанием (создание, редактирование, отмена тренировок), управление абонементами клиентов и просмотр общей статистики по клубу.

## Технологии

- **Frontend**: HTML5, CSS3, Vanilla JavaScript (ES6+).
- **Дизайн**: Адаптивный интерфейс (Mobile-First), современный дизайн с использованием парадигмы Glassmorphism, CSS-переменных и Flexbox/Grid.
- **Архитектура**: Клиентское приложение (SPA) с имитацией базы данных и состояния через локальные JavaScript-объекты (in-memory data).

## Модель данных (Классы и Сущности)

Система базируется на нескольких ключевых сущностях (классах), которые взаимодействуют друг с другом:

### 1. Пользователь (`User`)
Основной класс, представляющий участника системы.
- **Атрибуты**: `id`, `fullName`, `email`, `phone`, `role` (клиент, тренер, администратор), `password`, `avatar`.
- **Дополнительно для тренера**: `specialization`, `experience`.

### 2. Тренировка (`Training`)
Занятие в расписании фитнес-клуба.
- **Атрибуты**: `id`, `title`, `coachId` (связь с тренером), `date`, `time`, `duration`, `maxCapacity`, `currentBookings`, `type` (групповая/индивидуальная), `status`.

### 3. Абонемент (`Subscription`)
Купленный клиентом абонемент.
- **Атрибуты**: `id`, `clientId` (связь с клиентом), `type` (разовый, месячный и т.д.), `totalSessions`, `remainingSessions`, `startDate`, `endDate`, `status`, `price`.

### 4. Запись на тренировку (`Booking`)
Связующая сущность между клиентом и тренировкой.
- **Атрибуты**: `id`, `trainingId` (связь с тренировкой), `clientId` (связь с клиентом), `bookingDate`, `status`.

### 5. Тип абонемента (`SubscriptionType`)
Справочник доступных для покупки тарифов.
- **Атрибуты**: `id`, `name`, `sessions`, `duration`, `price`.

### Ассоциации (Связи между классами)
- **Клиент (1) — (M) Абонемент**: Один клиент может иметь историю из нескольких абонементов (но активен обычно один).
- **Тренер (1) — (M) Тренировка**: Один тренер проводит множество тренировок.
- **Клиент (M) — (M) Тренировка**: Связь реализуется через класс `Booking`. Один клиент записывается на разные тренировки, а на одной тренировке присутствует много клиентов.

## Как запустить

Поскольку приложение реализовано на чистом JavaScript без бэкенда, для его работы не требуется сложной настройки серверов.

1. Склонируйте репозиторий или скачайте архив.
2. Перейдите в папку `app` и откройте файл `index.html` в любом современном браузере.
3. Используйте следующие тестовые данные для входа в систему (для проверки разного функционала):

**Клиент** (управление абонементами, запись)
