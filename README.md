# 🏋️ FitClub — Сервис фитнес-клуба

> Веб-приложение для управления абонементами, записи на тренировки и учёта посещаемости фитнес-клуба.

## 📋 О проекте

Проект представляет собой одностраничное веб-приложение (SPA) для автоматизации ключевых процессов фитнес-клуба. Система обеспечивает взаимодействие трёх ролей: **клиентов**, **тренеров** и **администраторов**.

**Клиент:**
- Онлайн-запись на тренировки с отображением свободных мест
- Покупка абонемента (4 типа тарифов)
- Просмотр остатка занятий и истории посещений

**Тренер:**
- Просмотр личного расписания
- Электронная отметка присутствия клиентов

**Администратор:**
- Управление расписанием тренировок
- Управление абонементами клиентов
- Просмотр статистики и отчётов

## 🛠 Технологии

| Технология | Назначение |
|:---|:---|
| **HTML5** | Структура страниц |
| **CSS3** | Стилизация, адаптивность, CSS-переменные, Flexbox/Grid |
| **JavaScript (ES6+)** | Бизнес-логика, рендеринг, управление данными |

**Архитектура:** Клиентское приложение (SPA). Данные хранятся в JavaScript-объектах (in-memory) — серверная часть и база данных не используются.

## 🧩 Модель данных

### 1. Пользователь (`User`)
Участник системы с одной из трёх ролей.
- **Атрибуты**: `id`, `fullName`, `email`, `phone`, `role` (client / coach / admin), `password`, `avatar`.
- **Дополнительно для тренера**: `specialization`, `experience`.

### 2. Тренировка (`Training`)
Занятие в расписании фитнес-клуба.
- **Атрибуты**: `id`, `title`, `coachId`, `coachName`, `date`, `time`, `duration`, `maxCapacity`, `currentBookings`, `type` (групповая / индивидуальная), `status`, `icon`.

### 3. Абонемент (`Subscription`)
Купленный клиентом абонемент.
- **Атрибуты**: `id`, `clientId`, `type`, `totalSessions`, `remainingSessions`, `startDate`, `endDate`, `status`, `price`.

### 4. Запись на тренировку (`Booking`)
Бронирование места на тренировке.
- **Атрибуты**: `id`, `trainingId`, `clientId`, `bookingDate`, `status`.

### 5. Тип абонемента (`SubscriptionType`)
Справочник доступных тарифов.
- **Атрибуты**: `id`, `name`, `sessions`, `duration`, `price`, `icon`, `color`.

### Связи между сущностями
- **Клиент (1) → (M) Абонемент**: один клиент может иметь несколько абонементов (активен обычно один).
- **Тренер (1) → (M) Тренировка**: один тренер проводит множество тренировок.
- **Клиент (M) ↔ (M) Тренировка**: связь через `Booking` — клиент записывается на разные тренировки, на одной тренировке несколько клиентов.

## 🚀 Как запустить

Приложение работает на чистом JavaScript без серверной части — достаточно открыть HTML-файл в браузере.

1. Склонируйте репозиторий или скачайте архив.
2. Откройте файл `app/index.html` в любом современном браузере.
3. Используйте тестовые данные для входа:

| Роль | Email | Пароль |
|:---|:---|:---|
| **Клиент** | `anna@mail.ru` | `123456` |
| **Тренер** | `maria@mail.ru` | `123456` |
| **Администратор** | `admin@fitness.ru` | `admin` |
