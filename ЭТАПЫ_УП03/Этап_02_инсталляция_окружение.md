## 3. Docker-контейнеризация

### 3.1. Dockerfile

Используется `nginx:alpine` — легковесный веб-сервер для раздачи статических файлов.

```dockerfile
FROM nginx:alpine
LABEL maintainer="FitClub Engineering Team"
LABEL version="3.0"

COPY app/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
  CMD wget -qO- http://localhost/ || exit 1

CMD ["nginx", "-g", "daemon off;"]
```

**Обоснование выбора:**
- `nginx:alpine` — образ ~5 МБ, минимальная поверхность атаки
- `HEALTHCHECK` — автоматическая проверка состояния контейнера
- Собственный `nginx.conf` — заголовки безопасности и кэширование

### 3.2. Docker Compose

```yaml
version: '3.8'
services:
  fitclub:
    build: .
    container_name: fitclub-app
    ports:
      - "8080:80"
    restart: unless-stopped
```

### 3.3. Команды работы с Docker

| Команда | Описание |
|:---|:---|
| `docker build -t fitclub:latest .` | Сборка образа |

## 4. Скрипты запуска

### 4.1. Windows — `start.bat`

Скрипт автоматически определяет доступный способ запуска:
1. Python → `http.server` на порту 8000
2. Node.js → `npx serve` на порту 3000
3. Fallback → открытие `index.html` напрямую

### 4.2. Makefile (Linux/macOS)

| Команда | Описание |
|:---|:---|
| `make help` | Справка по командам |
| `make build` | Сборка Docker-образа |
| `make run` | Запуск контейнера |
| `make stop` | Остановка |
| `make test` | Запуск тестов |
| `make clean` | Очистка образов |

## 5. Конфигурация Nginx

Файл `nginx.conf` обеспечивает:
- **Gzip-сжатие** — уменьшение размера ответов
- **Заголовки безопасности** — X-Frame-Options, X-Content-Type-Options, CSP
- **Кэширование статики** — CSS/JS/изображения на 7 дней
- **SPA fallback** — перенаправление всех маршрутов на index.html
