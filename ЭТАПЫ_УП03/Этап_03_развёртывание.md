# Этап 3. Развёртывание итоговой версии

**Тема проекта:** Сервис фитнес-клуба (Абонементы, тренировки и посещаемость)  
**Дата выполнения:** 02.06.2026  
**Репозиторий:** [github.com/valtureso788/Fitness-03](https://github.com/valtureso788/Fitness-03)

---

## 1. Цель этапа

Развернуть итоговую версию приложения на GitHub Pages и в Docker-контейнере. Обеспечить доступность приложения по публичному URL.

---

## 2. Варианты развёртывания

| Способ | URL | Сложность | Для кого |
|:---|:---|:---|:---|
| Локально (файл) | `file:///app/index.html` | Минимальная | Быстрая проверка |
| Python HTTP Server | `http://localhost:8000` | Низкая | Разработка |
| Docker + Nginx | `http://localhost:8080` | Средняя | Продакшен-подобное |
| GitHub Pages | `https://valtureso788.github.io/Fitness-03/` | Низкая | Публичный доступ |

---

## 3. Развёртывание на GitHub Pages

### 3.1. Процесс

1. Код размещён в ветке `main` в папке `app/`
2. GitHub Actions автоматически публикует содержимое `app/` на Pages
3. Приложение доступно по URL: `https://valtureso788.github.io/Fitness-03/`

### 3.2. Настройка GitHub Pages

```
Settings → Pages → Source: GitHub Actions
```

### 3.3. Автоматический деплой (CI/CD)

Файл `.github/workflows/ci-cd.yml` содержит job `deploy`, который:
1. Прогоняет тесты
2. Проверяет синтаксис
3. Собирает Docker-образ
4. Публикует `app/` на GitHub Pages

---

## 4. Развёртывание в Docker

### 4.1. Сборка и запуск

```bash
# Сборка образа
docker build -t fitclub:latest .

# Запуск контейнера
docker-compose up -d

# Проверка
curl http://localhost:8080
```

### 4.2. Проверка работоспособности

| Проверка | Команда | Ожидаемый результат |
|:---|:---|:---|
| Контейнер запущен | `docker ps` | Статус `Up` |
| HTTP-ответ | `curl -I http://localhost:8080` | `200 OK` |
| Healthcheck | `docker inspect --format='{{.State.Health.Status}}' fitclub-app` | `healthy` |
| Логи | `docker logs fitclub-app` | Без ошибок |

---

## 5. Развёртывание локально (Windows)

### 5.1. Быстрый запуск

```
start.bat
```

Скрипт автоматически выбирает способ:
- Python → `python -m http.server 8000`
- Node.js → `npx serve app -l 3000`
- Fallback → открытие в браузере напрямую

### 5.2. Через Makefile (Linux/macOS)

```bash
make build  # Сборка Docker
make run    # Запуск
make stop   # Остановка
```
